     - URL: https://api.stlouisfed.org/fred/series/observations?series_id=...&api_key=YOUR_KEY

PART 2: FULL INTEGRATION ARCHITECTURE
================================================================================

  User Browser (React :3000)
       │ HTTP POST /api/claims
       ▼
  Backend (Node.js/Express :5000)
       │ 1. Auth middleware — verify JWT
       │ 2. Save claim to DB (status: 'pending')
       │ 3. POST http://localhost:5001/extract  → NLP Service
       │ 4. Receive {metric, value, year, confidence}
       │ 5. Query official_data_cache WHERE metric_name=? AND year=?
       │ 6. Calculate difference, percentage_error, verdict
       │ 7. INSERT into verification_log
       │ 8. UPDATE claims SET credibility_score, status='verified'
       │ 9. Return full verdict to frontend
       │
  ┌────┴────────────────┐
  │                     │
  ▼                     ▼
  NLP Service       MySQL Database
  (Python :5001)    (localhost:3306)
                    tables: users, claims,
                            official_data_cache,
                            verification_log

VERDICT LOGIC (goes in claimController.js):
  percentageError < 5%  → "accurate"
  percentageError < 20% → "misleading"
  percentageError >= 20% → "false"

PART 6: REGEX vs ML — UPGRADE PATH
================================================================================

  Phase 1 (NOW — MVP):
    ✅ Regex extraction (built)
    ✅ Claim probability scoring (built)
    [ ] Direct comparison against official_data_cache (needs backend seeder)

  Phase 2 (After ~1000 real claims collected):
    - Export claims from DB where human-reviewed verdict exists
    - Use as training data for SpaCy custom NER
    - Label: did regex get the right metric/value/year?
    - Fine-tune DistilBERT or SpaCy NER
    - Deploy as fallback when regex confidence < 0.6

  Phase 3 (Scale):
    - ML model becomes primary extractor
    - Regex becomes fast-path cache
    - If regex confidence >= 0.9 → skip ML, return immediately
    - If regex confidence < 0.9 → call ML model

================================================================================
BACKGROUND: THE THREE LAYERS OF CLAIM VERIFICATION
================================================================================

Layer 1 — EXTRACTION     → "What is being claimed?"
          (BUILT ✅)        regex → metric + value + year

Layer 2 — EVIDENCE        → "What does the real world say about this?"
          (NOT BUILT)       data APIs + news sources

Layer 3 — VERDICT         → "Does the evidence support or contradict the claim?"
          (NOT BUILT)       comparison logic + AI reasoning

The regex system handles Layer 1 completely.
The sophisticated model (RAV) is for Layers 2 and 3.

RAV = Retrieval-Augmented Verification
  → Don't train a model to memorize facts.
  → Instead, RETRIEVE evidence at query time and use a pre-trained
    reasoning model to compare the claim against that evidence.

================================================================================
THE THREE-TIER ENGINE
================================================================================

TIER 1 — Regex + Official Data API  (fast when cached; no ML)
  When: Claim has a clear numeric metric + year matching one of the 10 metrics
  What: Retrieve official value (World Bank first) and compute % error
  Verdict quality: High for well-formed numeric claims
  Cost: Free (World Bank API is free)

TIER 2 — Pre-trained NLI Model  (medium: 100-500ms, free, no training needed)
  When: Tier 1 gave low confidence OR claim is non-numeric
        e.g., "India's economy is booming"
  What: Retrieve 3-5 news snippets → run NLI model (entail/contradict/neutral)
  Model options (HuggingFace, no fine-tuning required for MVP):
    - facebook/bart-large-mnli            (recommended to start: simpler + faster to iterate)
    - cross-encoder/nli-deberta-v3-large  (upgrade later: higher quality, heavier download/CPU)
    Both are free, open-source, on HuggingFace. NO training needed.
    Load once, call at inference time.
  Verdict quality: Good for contextual/opinion claims
  Cost: Free (self-hosted, runs on CPU)

TIER 3 — LLM API  (slow: 1-3 seconds, best quality)
  When: Tier 2 confidence < 0.6 OR user requests deep analysis
  What: Send claim + evidence to Gemini/Grok with a structured prompt
  APIs to use:
    - Gemini 1.5 Flash (FREE: 15 req/min) — get key at aistudio.google.com
    - Grok API (xAI, paid but cheap, has real-time web search built in)
    - GPT-4o-mini (paid, ~$0.0001/claim — very cheap)
  Verdict quality: Highest — handles nuance, context, multi-source reasoning
  Cost: Free (Gemini) to very cheap ($0.001/claim GPT-4o-mini)

SECURITY NOTE (important for web app):
  Do NOT call Grok/Gemini from the browser (e.g., PuterJS in frontend) because API keys leak.
  Tier 3 should be called server-side (inside this FastAPI service or later via Node backend).

================================================================================
VERDICT ROUTER LOGIC
================================================================================

  if tier1_confidence > 0.8:
      return tier1 result immediately  ← regex + data API, fastest path

  elif tier1_confidence > 0.4:
      run tier2
      combine tier1 numeric check + tier2 NLI verdict
      return merged result

  else:
      run tier2
      if tier2_confidence < 0.6:
          escalate to tier3 (LLM API)
      return best available result

NEW PYDANTIC MODELS NEEDED IN main.py
================================================================================

  NumericCheckResult:
    official_value: float | None
    claimed_value: float | None
    percentage_error: float | None
    source: str | None
    indicator_code: str | None
    source_url: str | None

  EvidenceSnippet:
    source: str
    snippet: str
    url: str
    nli_verdict: str    ("entailment" | "contradiction" | "neutral")
    nli_score: float

  VerificationResult:
    claim: str
    tier_used: str      ("tier1" | "tier2" | "tier3")
    verdict: str        ("accurate" | "misleading" | "false" | "unverifiable")
    confidence: float
    numeric_check: NumericCheckResult | None
    evidence: list[EvidenceSnippet]
    explanation: str | None

PHASE 2 — FINE-TUNING (AFTER MVP, AFTER 1000+ REAL CLAIMS)
================================================================================

  Once you have real user claims with human-reviewed verdicts:

  1. Export from DB:
       SELECT claim_text, verdict FROM claims WHERE human_reviewed = TRUE

  2. Combine with existing datasets:
       LIAR dataset  — 12,800 labeled political claims
                        github.com/talhbutt/LIAR-PLUS
       FEVER         — 185,000 claims vs Wikipedia    fever.ai
       ClaimBuster   — US presidential debate claims  idir.uta.edu/claimbuster/

  3. Fine-tune DeBERTa or RoBERTa on your combined dataset:
       Model: microsoft/deberta-v3-base (good balance of size vs accuracy)
       Framework: HuggingFace Trainer API
       Hardware: Google Colab free GPU (T4) is sufficient for fine-tuning

  4. Deploy fine-tuned model as Tier 2 replacement:
       Old Tier 2: generic NLI model
       New Tier 2: your domain-specific model trained on Indian economic claims
